Investor Brief

YOLO 策略说明页

YOLO 不是一个“看到信号就冲”的粗糙高杠杆系统，而是一条完整的 先筛选、再打分、再确认、再执行、再保护、再退出 的交易链路。它的目标不是做长期低波动资产配置，而是在比赛窗口里，抓住最强的短中周期动量段，快速兑现收益。

这套策略适合向投资者解释的核心，不是“它会下很多单”，而是“它只在市场里挑最强的少数机会，而且每一个动作都有明确的量化门槛”。

用途：短窗口比赛策略 风格：高选择性动量追踪 执行：先 kit 下单，再做仓位保护 当前实盘：isolated + 硬止损单
One Sentence

一句话卖点

YOLO 的本质是：在全市场永续合约里，持续寻找方向一致、动量够强、成交量确认、资金费率有帮助的最强标的，只在最强标的满足入场确认时才动手，并用明确的止损、反转检测、时间退出和跟踪止盈来管理仓位。

对投资者最重要的表述是：YOLO 不是盲目高杠杆，它是高筛选强度下的高效率执行。
Why It Exists

为什么需要 YOLO

比赛窗口短

两周级别的比赛，不适合慢慢攒年化。策略必须优先抓短期最强的收益段。

市场机会分布不均匀

大部分时刻大部分币没有好机会，所以核心不是“经常交易”，而是“把最强机会挑出来”。

执行效率决定上限

如果识别到了机会，但杠杆、张数、止损、保证金模式没处理好，收益会直接被执行层吞掉。

风险必须可编程

YOLO 允许激进，但不能允许模糊。什么时候进、什么时候退、最多亏多少，都必须能落到参数和状态机里。

Framework

策略总流程

Scoring

第一步：先给全市场打分

YOLO 不是先看 gate。它的第一步，是先把动态 universe 里的所有永续合约都打一遍分，再只挑出最强的一个候选。也就是说，gate 是最后确认，不是第一层筛选。

评分在看什么

因子 权重 意义
ADX 35% 趋势强不强。没有趋势强度，动量往往走不远。
EMA Alignment 25% 短中周期均线是否朝同一方向排列。
Volume Ratio 20% 当前量能是否超过平时，判断是不是“真动”。
Funding Edge 10% 资金费率是否给当前方向带来额外帮助。
ATR Sweet Spot 10% 波动是不是在一个既能赚钱、又没过于失控的区间。

为什么先打分，再过 gate

  • 如果先对所有币直接套 gate，会把市场当成“符合 / 不符合”的二元世界，丢失强弱排序。
  • YOLO 的关键是“即使有多个都能做，也只做最强的那个”。
  • 这样做能把注意力集中到分数最高、结构最完整的合约，而不是同时开很多平庸单子。
  • 对投资者来说，这说明策略的核心不是频率，而是排序能力。
对外解释时可以直接说：YOLO 先做全市场 ranking，再做交易确认；不是任何出现信号的币都去碰。
Gates

第二步：选出最强后，再过 3 个 Gate

Gate 1: Trend Gate

目的：确认方向一致，而不是逆趋势硬冲。

当前口径：EMA alignment >= 60%

含义：如果要做多，就要求均线结构大体上支持多；如果要做空，就要求均线结构大体上支持空。

Gate 2: Momentum Gate

目的：确认这不是“看起来有方向，但实际上动能已经衰减”的假机会。

当前口径：

  • 多头 RSI：53 - 78
  • 空头 RSI：22 - 47
  • 同时要求 MACD histogram 不能明显反向

Gate 3: Volume Gate

目的：确认市场真的在参与，不是没人交易时的假突破。

当前口径：volume_ratio >= 1.1

含义：当前成交量至少要比自己的平均水平高 10% 以上。

对投资者最容易理解的一句话是：分数告诉我们“谁最强”，3 个 gate 告诉我们“现在能不能做”。
Leverage

第三步：杠杆不是固定值，而是按波动决定

先按 ATR 决定目标杠杆

ATR% 目标杠杆 解释
< 1% 75x 波动相对低，可以给更高效率。
1% - 3% 50x 中等波动，对应默认主档。
> 3% 30x 波动太大，主动降杠杆。

但真正执行前还要再裁两刀

  • 先看交易所该合约允许的 最大杠杆。
  • 再看账户在当前保证金模式下允许的 最大可下张数。
  • 所以最终执行的不是“理想杠杆”，而是 effective leverage。

这也是为什么页面里现在会同时显示 Max Lev 和 Eff Lev。策略不是死板地按 50x 下，而是先服从交易所和账户约束。

Entry

什么时候真正开仓

步骤 1
选出 Top 1
全市场分数最高
步骤 2
过 3 Gate
趋势、动量、量能同时通过
步骤 3
算杠杆和张数
ATR 决定，交易所再裁剪
步骤 4
真正下单
kit 开仓 + 止损 + 逐仓保护

执行层现在做了什么

  • 用 kit 下开仓单，满足比赛要求。
  • 默认使用 isolated，不再把整池 USDT 作为同一个 cross 风险池。
  • 开仓时直接附带硬止损单。
  • 开仓成功后，再通过原生 API 给该 isolated 仓位补保证金。

为什么这很重要

  • 投资者最怕的不是波动，而是“看起来只下 50U，实际上把 300U 都拖进风险池”。
  • 逐仓 + 附带止损 + 补保证金，把风险从“账户级联动”收缩成“单仓位风险桶”。
  • 这让 YOLO 虽然激进，但不是无边界激进。
Exit

什么时候关仓

Target Hit

达到本轮目标利润，直接兑现。这是最理想的退出。

Hard Stop

达到本轮硬止损阈值时退出。现在同时有本地逻辑和交易所附带止损单两层保护。

Trailing Stop

一旦盈利形成，就启动跟踪止盈，把“浮盈”变成“落袋”。

Reversal Detection

如果原先支持交易的动量结构开始反转，策略会主动收紧，严重时直接离场。

Time Decay

如果仓位拖太久却没有走出来，说明原本的短周期优势已经衰减，应该退出。

Position Safety

如果交易所仓位状态与本地状态不一致，系统会优先以交易所真实仓位为准，避免幽灵单。

Capital Model

资金管理：YOLO 不是一次性全押，而是轮次推进

这套结构要怎么向投资者解释

  • YOLO 不是从第一笔就把所有资金打进去，而是按轮次推进。
  • 第一笔只是最小试探资金桶。
  • 只有在前一轮失败并确认要进入下一轮时，才会提升资金占用。
  • 如果账户可用资金不足以支持下一轮，策略直接进入 RECHARGE_REQUIRED，不会盲开。
Risk

投资者最关心的风险怎么回答

主要风险

  • YOLO 本质上仍然是高波动、高效率的比赛策略，不适合被包装成低回撤稳健配置。
  • 极端跳空、流动性真空、异常插针，仍然可能让止损成交价格劣于理想值。
  • 这条策略的收益分布，不会像低频套利一样平滑，而是依赖少数高质量动量单。

但我们已经补了哪些保护

  • 从 cross 改成 isolated。
  • 开仓同时附带硬止损单。
  • 开仓后自动补逐仓保证金。
  • 杠杆先按 ATR 选，再按交易所上限和账户 max-size 裁剪。
  • 资金不够支持下一轮时，不再继续交易，而是停机等补资金。
适合对外说的版本是：YOLO 不是“无脑高杠杆”，而是“在严格过滤后的高效率出击，并且执行层已经内置隔离和保护”。
Pitch

一段可以直接拿去讲的投资者版本

YOLO 的核心不是交易频率，而是排序能力。它会先在全市场永续合约里持续打分，只选出当前最强的那一个候选；然后再要求这个候选同时满足趋势、动量和成交量三层确认。只有在“最强”且“确认成立”时，策略才会进入执行环节。

执行时，YOLO 不会盲目使用固定杠杆，而是先根据波动率决定目标杠杆，再根据交易所实际允许的杠杆和账户当前可下张数做裁剪。开仓后，系统同时做逐仓隔离、附带硬止损、补逐仓保证金和动态退出管理，确保它不是裸奔式高杠杆，而是一套有明确风险边界的比赛型高效率策略。

所以这套系统的 edge，不是一个神奇指标，而是：全市场排名 + 多层 gate 确认 + 真实可执行的风险控制。
